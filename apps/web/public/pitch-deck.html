AgentVault 02
THE PROBLEM

AI agents need to act financially.
Current models are broken.

🔒

No Access

AI agents can reason about financial decisions but can't execute them. They're stuck asking humans to click buttons.

🔑

Full Key Access

Giving agents your private key = unlimited risk. One bug, one prompt injection, and your entire wallet is drained.

⚠️

Hot Wallets

Custodial wrappers with unrestricted permissions. No scoping, no limits, no audit trail. A liability.

Powerful agents without boundaries are a liability.

AgentVault 03
THE SOLUTION

Agents with limits.

AgentVault is a permissioned execution layer that lets AI agents safely interact with paid APIs and on-chain workflows on Hedera — without ever accessing a user's private keys.

CONTROL

Scoped Permissions

Session keys with explicit limits: which contracts, which methods, how much value, how long. Principle of least privilege.

EARN

API Marketplace

Turn any API into a paid service. Set your price, agents pay per request. Usage-based economics.

AI READY

MCP Integration

Expose services as MCP servers. AI agents like Claude and ChatGPT discover and use them natively.

Autonomous execution, without autonomous risk.

AgentVault 05

Core Architecture

Five primitives that work together to enable bounded agent autonomy.

LAYER 1

Smart Account

Standard EOA upgraded to enforce session keys and scoped permissions on-chain. Primary key never shared.

LAYER 2

Session Keys

Time-bounded, target-restricted keys. Define allowed contracts, methods, asset limits, and expiry.

LAYER 3

API Proxies

Any API wrapped as a paid service with encrypted headers, variable substitution, and usage tracking.

LAYER 4

Workflow Engine

Multi-step automations combining HTTP API calls, data transforms, and on-chain transactions into reusable templates.

LAYER 5

MCP Servers

Model Context Protocol servers that expose services and automations for discovery by AI agents like Claude and ChatGPT.

AgentVault 06
HEDERA INTEGRATION

Deeply Integrated with Hedera

AgentVault uses 5 Hedera services across the full stack.

⛓

HSCS — Smart Contracts

AgentDelegator contract deployed on Hedera Testnet. Enforces session key permissions, manages grants and revocations on-chain.

0x624f7c953dac044f3a38e7230c16f410cf7301d2
📜

HCS — Audit Trail

Every workflow execution, payment, and tool invocation is logged to HCS topics. Queryable via Mirror Node REST API.

🤖

HCS-10 — Agent Identity

MCP server registered as HCS-10 compliant agent on Hashgraph Online registry with inbound/outbound/profile topics.

💰

HTS — Token Payments

USDC payments via HTS-compatible ERC-20 transfers. Session keys enforce per-request payment limits.

🔍

Mirror Node — Verification

HCS messages and transaction history queried via Mirror Node REST API. HashScan links for on-chain verification.

⚡

Hedera EVM — Execution

executeWithSession relay via Hedera EVM. wagmi/viem configured with hederaTestnet chain. Gas-efficient execution.

AI IN ACTION

Claude Discovers & Uses AgentVault Tools

Real MCP integration — Claude autonomously discovers and calls AgentVault services.

User: "What's the current price of HBAR and look up account 0.0.1022530 on Hedera testnet" Claude: I'll use two AgentVault tools for this. Tool call: mcp__agentvault__token_price_api tokenId: "hedera-hashgraph" Result: { usd: 0.0926, usd_24h_change: 3.01% } Tool call: mcp__agentvault__hedera_account_balance accountId: "0.0.1022530" Result: { balance: 100 HBAR, tokens: [...] } Claude: HBAR is trading at $0.0926 (+3.01%). Account 0.0.1022530 holds 100 HBAR (~$9.26).

What's Happening

  • Claude connects via MCP stdio transport
  • Tools are loaded dynamically from the database
  • Proxy fetches live data from CoinGecko & Mirror Node
  • Session key limits what the agent can access
  • Every call logged to HCS audit trail
  • No API keys or private keys shared with the AI

This is not a mock. These are real API calls executed by Claude through AgentVault's MCP server.
